# LuxuryStay HMS Roadmap

## Vision

LuxuryStay HMS will become a secure, modern, scalable hotel management platform aligned with the official eProject brief for Software Engineering. The system should feel production-grade in workflow design, UI/UX quality, code structure, and documentation.

## Functional Coverage Map

### Completed Foundation

- role-based authentication
- public room listing and room detail pages
- room and room-type CRUD
- reservation creation
- receptionist check-in/check-out flow
- housekeeping dashboard
- maintenance dashboard
- notification system

### In Progress

- stronger authorization rules on backend routes
- environment-based API/socket configuration
- upload contract consistency
- project documentation cleanup

### Pending Major Modules

- guest profile persistence with editable preferences and identity details
- feedback and ratings with analytics-ready storage
- advanced billing with configurable taxes, services, and payment summaries
- admin system settings for rates, policies, and service rules
- occupancy, revenue, and service analytics dashboards
- monitoring, backups, audit logs, and QA workflows

## Delivery Phases

### Phase 1: Foundation Hardening

- secure all role-sensitive API routes
- normalize frontend/backend API contracts
- centralize runtime environment configuration
- document setup, architecture, and delivery scope
- stabilize linting and code quality baseline

### Phase 2: Core Hospitality Operations

- complete guest profile CRUD
- improve reservation validation and booking lifecycle
- enrich check-in/check-out with billing details
- support maintenance reporting from guest and staff channels
- make staff status management persistent

### Phase 3: Management Intelligence

- occupancy trends
- revenue analytics
- service performance metrics
- maintenance SLA views
- guest feedback reporting

### Phase 4: Production Readiness

- automated tests
- audit logging
- backup and recovery notes
- deployment guide
- user manual
- developer documentation
- demo video script and feature checklist

## UX Direction

- premium hospitality visual language
- fully responsive layouts
- fast, low-friction staff workflows
- clear status colors and live operational awareness
- polished role dashboards with meaningful summaries

## Success Criteria

- secure role-based access
- accurate room, booking, and billing state transitions
- smooth guest and staff workflows within 1 to 2 seconds for common actions
- professional academic documentation
- clean, modern UI suitable for a final-year showcase
